================================================================================
POSSIVEIS CONFLITOS COM API
================================================================================

NAO ha mudancas na comunicacao com a API. Todas as alteracoes sao puramente
de frontend/UI. Os seguintes pontos permanecem inalterados:

1. Endpoints utilizados:
   - GET /client/bitrefill/catalog/countries
   - GET /client/bitrefill/catalog/categories  
   - GET /client/bitrefill/catalog/products
   - POST /client/bitrefill/orders
   - POST /client/boltz/orders
   - GET /client/orders/{orderId}
   - GET /client/boltz/orders/{orderId}

2. Estrutura de dados esperada:
   - Paises: { data: [{ code, name }] }
   - Categorias: { data: [{ slug, label }] }
   - Produtos: { products: [{ id, name, currency, in_stock, packages, ... }] }
   - Ordens: mantém estrutura original (order_id, status, deposit_btc_address, etc.)

3. Polling de status: continua funcionando normalmente

4. Estados de ordem: mapeamento permanece o mesmo
   - Boltz: awaiting_deposit -> deposit_detected -> lockup_pending -> 
            provider_claim_pending -> paid_out
   - Bitrefill: awaiting_deposit -> deposit_detected -> confirming -> 
                provider_claim_pending -> paid_out
